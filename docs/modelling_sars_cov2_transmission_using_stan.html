Modelling SARS-CoV-2 transmission using Stan¶

Disclaimer: I've studied computer science and computational biology, not epidemiology or infectious disease biology. Hence, I'd like to ask you to not use this document as a reference whatsoever. It only serves as a small study for me to use some epidemiological models and ODEs in Stan. There are many capable epidemiologists in Switzerland whose papers, blogs and statements are far better references than this notebook.

In this notebook, we use a susceptible-exposed-infectious-recovered (SEIR) model, to model transmission of SARS-CoV-2 in Switzerland. Following Maia Martcheva's book and Wikipedia the SEIR model consists of a set of ordinary differential equations:

\begin{align} \frac{\delta S}{\delta t} &= - \theta_1 \frac{I}{N}S\\ \frac{\delta E}{\delta t} &= \theta_1 \frac{I}{N}S - \theta_2 E \\ \frac{\delta I}{\delta t} &= \theta_2 E - \theta_3 I \\ \frac{\delta R}{\delta t} &= \theta_3 I \end{align}

where $S$ is the number of suspectible individuals in a population (all in the beginning), $E$ is the number of exposed individuals, i.e., those who are infected but cannot transmit the disease yet, $I$ is the number of infected people, and $R$ the number of recovered ones. The parameters $\theta_1, \theta_2, \theta_3$ model the rates between the different states.

Here, we want to estimate the parameters $\boldsymbol \theta$ using a time-series of the numbers of infected people. We do it with R, and Stan in particular.

Stan model¶

Specifying the above ODE in Stan is fairly straightforward. Since it is generally easier to work with continuous data than discrete data, we model the stages as fractions, i.e., normalized by the population size. Furthermore, we also try to estimate the initial number of susceptible people using a latent variable $s_0$.

In [3]:
seir.model.file <- "_models/seir_compartment_model.stan"
cat(readLines(seir.model.file), sep = "\n")

functions {
  real[] seir(real t, real[] y, real[] theta, real[] x_r, int[] x_i) {
    real dydt[4];
    dydt[1] = -theta[1] * y[1] * y[3];
    dydt[2] = theta[1] * y[1] * y[3] - theta[2] * y[2];
    dydt[3] = theta[2] * y[2] - theta[3] * y[3];
    dydt[4] = theta[3] * y[3];
    return dydt;
  }
}

data {
  int<lower=1> n;
  int<lower=1> n_population;
  int y[n];
  real t0;
  real t[n];
}

transformed data {
  real x_r[0];
  int x_i[0];
}

parameters {
  real<lower=0> theta[3];
  real<lower=0, upper=1> s0;
}

transformed parameters {
  real y0[4] = {s0, 0.0, 1.0 - s0, 0.0};
}

model {
  real lambda[n];
  real y_hat[n, 4];
  theta ~ gamma(3, 1);
  s0 ~ beta(0.5, 0.5);

  y_hat = integrate_ode_rk45(seir, y0, t0, t, theta, x_r, x_i);

  for (i in 1:n)
    lambda[i] = y_hat[i, 3] * n_population;
  y ~ poisson(lambda);
}

generated quantities {
  real R0 = theta[1] / theta[3];
}

The first block, the function called seir, is, you guessed it, the SEIR model. It is a one-to-one translation of the mathematical formulation. The following blocks specify input data, parameters and the observation model. In the model block we solve the ODE using integrate_ode_rk45. In the last block we estimate the basic reproduction number $R_0$.

Testing the model on simulated data¶

Before we use the SEIR model on case numbers of the canton Basel-Stadt in Switzerland, we test if it works. For that we first simulate data. We can also use Stan for this. Note that here, I directly work on integers, just for convenience.

In [12]:
seir.gen.data.file <- "_models/seir_compartment_model_generate_data.stan"
cat(readLines(seir.gen.data.file), sep = "\n")

functions {
  real[] seir(real t, real[] y, real[] theta, real[] x_r, int[] x_i) {
    real dydt[4];
    real N = y[1] + y[2] + y[3] + y[4];
    dydt[1] = -theta[1] * y[1] * y[3] / N;
    dydt[2] = theta[1] * y[1] * y[3] / N - theta[2] * y[2];
    dydt[3] = theta[2] * y[2] - theta[3] * y[3];
    dydt[4] = theta[3] * y[3];
    return dydt;
  }
}

data {Q
  int<lower=0> n;
  real<lower=0> n_population;
  real t0;
  real t[n];
  real theta[3];
}

transformed data {
  real x_r[0];
  int x_i[0];
  real y0[4] = {n_population - 1.0, 0.0, 1.0, 0.0};
}

generated quantities {
  real y_hat[n, 4];
  y_hat = integrate_ode_rk45(seir, y0, t0, t, theta, x_r, x_i);
}

Note that we don't need a model block here, we only specify the SEIR model and initial conditions, and then solve the ODEs. So let's simulate some data.

Looking at the numbers, it's obvious that our model cannot deal with this, since it looks as if the transmission rates change over time (for instance due to social distancing measures). Hence, we would need to model transmission as a time-dependent variable $\beta_t$. However, let's first try this simple model only the last couple of days. The "ausserordentliche Lage" of the Swiss government began earlier than this, so we could assume that the transmission is a constant.

Time-dependent $\beta_t$¶

The next step would be to allow $beta$ to change with time, in order to model the decrease of transmission rates, for instance due to social distancing or the like. We could model this, for instance, using a spline piece-wise linear spline.
